# Security Policy

## Supported Versions

Only the latest stable release receives security fixes.

| Version | Supported          |
|---------|--------------------|
| 1.x     | ✅ Yes             |
| 0.x     | ❌ No (beta only)  |

## Reporting a Vulnerability

**Please do NOT report security vulnerabilities through public GitHub Issues.**

### How to report

Send an email to **omdasoft@hotmail.com** with the subject:

```
[SECURITY] laravel-webauthn — brief description
```

Include as much of the following as possible:

- A clear description of the vulnerability
- The affected version(s)
- Steps to reproduce or a proof-of-concept
- The potential impact (e.g., authentication bypass, data exposure)
- Any suggested mitigations

### What to expect

| Step | Timeline |
|---|---|
| Acknowledgement | Within 48 hours |
| Triage and severity assessment | Within 5 days |
| Security patch released | Within 14 days (critical issues within 7 days) |
| Public disclosure | After patch is released and users have had time to upgrade |

We follow [Coordinated Vulnerability Disclosure](https://www.cisa.gov/coordinated-vulnerability-disclosure-process).

### Credit

If you report a valid vulnerability, we will credit you in the release notes and `CHANGELOG.md` unless you prefer to remain anonymous.
